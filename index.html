Help & overview
For PMs & admins
What is this tool? ▾

Runway gives your team a consistent, repeatable way to evaluate and prioritize incoming feature and project requests.

  • Score every request against the same weighted criteria — no more gut-feel decisions
  • Track what came in, what was accepted, and what was passed on
  • Communicate decisions back to submitters with a built-in email workflow
New request tab ▾

Use this tab to score a request yourself — e.g. when a rep walks you through something verbally.

  • Select a project type first — this controls which criteria appear and which weight preset is applied
  • Four fields are always required: Project / feature name, Customer / prospect, Submitted by, and Submitter email
  • Fill in any additional detail fields and answer each criterion
  • Save to add the project to the tracker with a full score and tier
Dashboard ▾

A running list of every scored request. Filterable by status, project type, tier, date range, and rep.

  • Click any row to expand it — view full submission details, per-criterion scores, and decision controls
  • Update status, set a revisit date, and add decision notes — all save instantly
  • Send a decision email directly from the panel with a live preview
  • Use Edit & re-score to pre-fill the intake form and update the project in place — a score trend indicator (↑↓) shows if the score improved or declined
  • Switch to Board view for a kanban layout with drag-and-drop status changes
  • Export as CSV or import a submission using the import code from an email
Status & decision workflow ▾

Every project moves through: Submitted → Under Review → Accepted → Completed / Deferred / Passed.

  • Change status from the expanded row or by dragging cards on the board — updates instantly
  • Once accepted and shipped to Jira, mark as Completed to close the loop — Completed counts toward decided metrics and appears in the Insights charts
  • For deferred projects, set a revisit date; the Summary metrics card turns red/amber when overdue or due within 7 days — click it to jump straight to those projects
  • Click Send decision email to open a modal with a live preview — adjust decision, notes, and revisit date before sending
  • Use Edit & re-score to update an existing project without losing its status or decision history
Scoring weights tab ▾

Control how much each criterion counts toward the final score.

  • Use weight sliders to adjust the relative importance of each criterion
  • Apply a preset to instantly shift the weighting profile (e.g. revenue-focused vs. strategic fit)
  • Save your own custom presets for reuse
  • Map a preset to each project type — weights shift automatically when a type is selected
  • Control which criteria are visible per project type, hiding irrelevant questions entirely
Edit criteria tab ▾

Customize the questions, answer options, and detail fields that appear on the intake form.

  • Add, remove, reorder, and rename scoring criteria and their answer options
  • Customize the project detail fields — add dropdowns, text fields, or long-form inputs
  • Use the Hint field on each criterion for in-context guidance shown below the question
  • Once set up, use the Scoring weights tab to control which criteria appear per project type
Exporting the sales form ▾

From the Edit criteria tab, click Export sales form to generate a standalone HTML file for Sales and CS reps.

  • The export modal confirms your email address and CC — all EmailJS credentials come from Settings automatically
  • Reps must select a project type; only the relevant criteria for that type are shown
  • Scoring, weights, and tier logic are completely hidden from reps
  • Generate personalized rep links from Settings → Rep links — the rep's name and email pre-fill and lock automatically when they open their link
  • Three email notifications are available — all use pre-built HTML, no writing required: Intake notification (to PM), Submission auto-reply (to rep, optional), and Decision notification (manual from All Projects)
Dashboard & board view ▾

The Dashboard has two views — List and Board — plus a summary metrics panel at the top.

  • The Summary metrics panel shows total projects, submissions this month, average score, acceptance rate, and overdue revisits — click any card to filter the list
  • Switch to Board view for a five-column kanban layout — drag cards between columns to update status instantly
  • Terminal columns (Accepted, Completed, Deferred, Passed) collapse by default to keep focus on active work — click the column header to expand or collapse
  • Sort board cards by score or date using the toolbar above the board
  • Filter board cards by project type using the type buttons in the toolbar
  • Overdue deferred projects are highlighted with a red left border accent in both list and board views
Insights tab ▾

The Insights tab shows charts and tables built from your project data, with filters to scope everything to a specific time range, type, rep, or status.

  • Six charts: submissions over time, pipeline by status, score distribution, avg score by rep, acceptance rate by type, and avg time to decision
  • Criteria scoring breakdown table — avg score per criterion across all projects, sorted highest to lowest. Useful for spotting where requests consistently score weak
  • Top customers table — submission volume, avg score, and acceptance rate per account
  • Filter bar at the top scopes all charts and tables simultaneously — use presets (last 30/90 days, etc.) or set a custom date range
  • Click Export report to generate a shareable HTML report that opens in a new tab and can be printed to PDF
Submitters tab ▾

The Submitters tab aggregates all submissions by rep, giving you a quick read on who's submitting what.

  • Each rep card shows their total submissions, average score, acceptance rate, and last submission date
  • A color-coded status bar shows the breakdown of their submissions across all five statuses
  • Sort cards by submission count, average score, or acceptance rate
  • Click any rep card to jump to All Projects filtered to just their submissions
  • Generate personalized intake form links per rep from Settings → Rep links — their name and email pre-fill and lock on the form so attribution is always accurate
Backing up and restoring settings ▾

All data lives in your browser's local storage. The Settings tab is the central place for everything administrative.

  • Configure all three EmailJS template IDs and email addresses in one place
  • Generate and manage personalized rep links under Rep links
  • Use Generate sample projects to seed the tracker with test data — Clear sample projects removes only the generated ones
  • Export or import a full settings backup (criteria, weights, presets, thresholds, help content)
  • Separately export or restore a JSON backup of saved projects
  • Save as defaults sets a local restore point — Reset to defaults returns to your saved configuration, not the original built-ins
