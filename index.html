Help & overview
For PMs & admins
What is this tool? ▾

The Project Intake Scorecard gives your team a consistent, repeatable way to evaluate incoming feature and project requests.

  • Score every request against the same weighted criteria — no more gut-feel decisions
  • Track what came in, what was accepted, and what was passed on
  • Communicate decisions back to submitters with a built-in email workflow
New request tab ▾

Use this tab to score a request yourself — e.g. when a rep walks you through something verbally.

  • Select a project type first — this controls which criteria appear and which weight preset is applied
  • Four fields are always required: Project / feature name, Customer / prospect, Submitted by, and Submitter email
  • Fill in any additional detail fields and answer each criterion
  • Save to add the project to the tracker with a full score and tier
All projects tab ▾

A running list of every scored request, filterable by status. Click any row to expand it.

  • View full submission details and per-criterion scores with weight percentages
  • Update status, set a revisit date, and add decision notes
  • Send a decision email directly from the panel — includes a live preview
  • Use Edit & re-score to pre-fill the intake form and update the project in place
  • A banner flags deferred projects that are overdue or due within 7 days
  • Export as CSV or import a submission using the import code from an email
Status & decision workflow ▾

Every project moves through: Submitted → Under Review → Accepted / Deferred / Passed.

  • Change status from the expanded row — the badge updates instantly and a toast confirms
  • For deferred projects, set a revisit date; a reminder banner appears when it's due or within 7 days
  • Click Send decision email to open a modal with a live preview — adjust notes and date before sending
  • Use Edit & re-score to update an existing project without losing its status or decision history
Scoring weights tab ▾

Control how much each criterion counts toward the final score.

  • Use weight sliders to adjust the relative importance of each criterion
  • Apply a preset to instantly shift the weighting profile (e.g. revenue-focused vs. strategic fit)
  • Save your own custom presets for reuse
  • Map a preset to each project type — weights shift automatically when a type is selected
  • Control which criteria are visible per project type, hiding irrelevant questions entirely
Edit criteria tab ▾

Customize the questions, answer options, and detail fields that appear on the intake form.

  • Add, remove, reorder, and rename scoring criteria and their answer options
  • Customize the project detail fields — add dropdowns, text fields, or long-form inputs
  • Use the Hint field on each criterion for in-context guidance shown below the question
  • Once set up, use the Scoring weights tab to control which criteria appear per project type
Exporting the sales form ▾

From the Edit criteria tab, click Export sales form to generate a standalone HTML file for Sales and CS reps.

  • The export modal confirms your email address and CC — all EmailJS credentials come from Settings automatically
  • Reps must select a project type; only the relevant criteria for that type are shown
  • Scoring, weights, and tier logic are completely hidden from reps
  • Three email notifications are available — all use pre-built HTML, no writing required:
  • Intake notification — sent to the PM automatically when a rep submits
  • Submission auto-reply — sent to the rep immediately confirming receipt (optional)
  • Decision notification — sent manually from All Projects when a decision is ready
Backing up and restoring settings ▾

All data lives in your browser's local storage. The Settings tab is the central place for everything administrative.

  • Configure all three EmailJS template IDs and email addresses in one place
  • Export or import a full settings backup (criteria, weights, presets, thresholds, help content)
  • Separately export or restore a JSON backup of saved projects
  • Save as defaults sets a local restore point — Reset to defaults returns to your saved configuration, not the original built-ins
